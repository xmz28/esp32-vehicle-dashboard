# 485 CAN Android 仪表固件用户使用说明

## 1. 文档用途

本文档面向拿到 `.bin` 固件并负责硬件安装、接线和使用的人员。

本文只说明：

- 所需硬件
- ESP32 IO 定义
- 外设接线
- 基本操作
- 固件主要功能
- 烧录和 OTA 升级注意事项

不包含程序实现原理、通信报文格式和车辆协议说明。

当前配套固件：

| 项目 | 内容 |
|---|---|
| 固件版本 | `0.1.5` |
| 固件文件 | `ota_bins/firmware-0.1.5.bin` |
| 适用主控 | ESP32 Dev Module / ESP-WROOM-32 |
| Flash 要求 | 建议 4 MB |
| 屏幕 | ST7789，240 × 240 |

## 2. 主要硬件

建议准备以下硬件：

1. ESP32 DevKit / ESP-WROOM-32 开发板
2. 240 × 240 ST7789 SPI 屏幕
3. RS485 收发模块
4. 3.3V CAN 收发器，例如 SN65HVD230
5. PCF8575 IO 扩展模块，地址设置为 `0x20`
6. 三个操作按键：UP、DOWN、OK
7. 可选 MAX98357A I2S 功放和扬声器
8. 必要的电源、电平转换、保险和接口保护电路

所有模块必须共地。

> 警告：车辆上的 12V、48V、60V 等电源或信号不得直接接入 ESP32、PCF8575 或其他 3.3V IO。必须经过降压、电平转换、隔离或保护电路。

## 3. ESP32 IO 总表

| GPIO | 功能 | 方向 | 连接对象 |
|---:|---|---|---|
| 0 | KEY0 / BOOT 快捷键 | 输入，低有效 | 板载 BOOT 键或外接按键 |
| 2 | WiFi 客户端指示灯 | 输出，高有效 | LED，需串联限流电阻 |
| 4 | RS485 RE/DE | 输出 | RS485 模块 RE 与 DE |
| 5 | TFT CS | 输出 | ST7789 CS |
| 13 | I2S LRC / WS | 输出 | MAX98357A LRC |
| 14 | I2S BCLK | 输出 | MAX98357A BCLK |
| 15 | TFT BL | PWM 输出 | ST7789 背光控制 |
| 16 | RS485 RX | 输入 | RS485 模块 RO |
| 17 | RS485 TX | 输出 | RS485 模块 DI |
| 18 | TFT SCLK | 输出 | ST7789 SCL/SCK |
| 19 | 电源解锁控制 | 输出，高有效 | 外部电源控制电路 |
| 21 | CAN TX | 输出 | CAN 收发器 CTX/TXD |
| 22 | CAN RX | 输入 | CAN 收发器 CRX/RXD |
| 23 | TFT MOSI | 输出 | ST7789 SDA/MOSI |
| 25 | I2S DATA | 输出 | MAX98357A DIN |
| 26 | TFT RST | 输出 | ST7789 RST/RES |
| 27 | TFT DC | 输出 | ST7789 DC/A0 |
| 32 | I2C SDA | 双向 | PCF8575 SDA |
| 33 | I2C SCL | 输出 | PCF8575 SCL |

### 重要说明

- GPIO15 是当前固件的屏幕背光控制脚，不再把 ST7789 的 BL 直接接到 3.3V。
- GPIO19 只能作为控制信号使用，不能直接带动大电流负载。控制继电器、MOS 管或整车电源时必须增加驱动和保护电路。
- GPIO2 外接 LED 时必须串联限流电阻。
- GPIO0 是 ESP32 启动配置脚，外接电路不能在上电时长期强制拉低，否则可能进入下载模式。

## 4. ST7789 屏幕接线

当前程序按 240 × 240 SPI 屏幕配置。

| ST7789 | ESP32 | 说明 |
|---|---|---|
| GND | GND | 共地 |
| VCC | 3.3V | 以屏幕模块标称电压为准 |
| SCL / SCK | GPIO18 | SPI 时钟 |
| SDA / MOSI | GPIO23 | SPI 数据 |
| RST / RES | GPIO26 | 屏幕复位 |
| DC / A0 | GPIO27 | 数据/命令 |
| CS | GPIO5 | 片选 |
| BL / LED | GPIO15 | PWM 背光控制 |

本项目不使用 TFT 的 MISO/SDO。

部分屏幕模块的背光电流较大，GPIO15 应只接模块的背光控制端。若模块没有背光驱动电路，应增加三极管或 MOS 管，禁止让 ESP32 IO 直接承担大电流。

## 5. RS485 接线

程序使用 ESP32 UART2。

| RS485 模块 | ESP32 | 说明 |
|---|---|---|
| RO | GPIO16 | ESP32 接收 |
| DI | GPIO17 | ESP32 发送 |
| RE | GPIO4 | 与 DE 并接 |
| DE | GPIO4 | 与 RE 并接 |
| GND | GND | 必须共地 |
| A | RS485 A | 接总线 A |
| B | RS485 B | 接总线 B |

默认串口参数：

- 波特率：115200
- 数据格式：8N1

注意事项：

- 优先使用适合 3.3V IO 的 RS485 收发器。
- 如果使用 5V MAX485 模块，必须确认模块的 RO 输出不会向 ESP32 输入 5V；必要时增加电平转换。
- A、B 接反时通常无法正常接收，可断电后互换检查。
- 位于总线末端时，根据现场总线要求配置 120Ω 终端电阻。
- RS485 总线建议使用双绞线。

## 6. CAN 接线

ESP32 的 GPIO21、GPIO22 不能直接连接 CANH、CANL，必须经过 CAN 收发器。

以 SN65HVD230 为例：

| SN65HVD230 | ESP32 / 总线 |
|---|---|
| CTX / TXD | GPIO21 |
| CRX / RXD | GPIO22 |
| VCC | 3.3V |
| GND | GND |
| CANH | CAN 总线 CANH |
| CANL | CAN 总线 CANL |

默认 CAN 波特率为 500 kbit/s。

注意事项：

- CANH、CANL 不可接反。
- 总线两端通常各使用一个 120Ω 终端电阻，不能在每个节点都重复增加。
- 建议使用双绞线，并尽量缩短支线。
- 车辆环境中建议增加浪涌、静电和反接保护。

## 7. PCF8575 与硬线输入

### 7.1 I2C 接线

| PCF8575 | ESP32 |
|---|---|
| SDA | GPIO32 |
| SCL | GPIO33 |
| GND | GND |
| VCC | 建议 3.3V |

PCF8575 地址必须设置为 `0x20`。

如果 PCF8575 模块使用 5V 供电，必须确认 SDA、SCL 上拉电压不会超过 ESP32 的允许范围；建议使用 3.3V 供电或增加双向电平转换。

### 7.2 PCF8575 输入定义

| PCF8575 端口 | 程序位 | 功能 | 有效状态 |
|---|---:|---|---|
| P03 | 3 | 左转向灯输入 | 高电平有效 |
| P04 | 4 | 右转向灯输入 | 高电平有效 |
| P05 | 5 | 位置灯输入 | 高电平有效 |
| P06 | 6 | 近光灯输入 | 高电平有效 |
| P07 | 7 | 远光灯输入 | 高电平有效 |
| P13 | 11 | 外部 DC / 屏幕唤醒检测 | 当前程序低电平表示外部 DC 存在 |
| P14 | 12 | 整车蓝牙指示输入 | 低电平有效 |
| P15 | 13 | OK / MENU 按键 | 低电平有效 |
| P16 | 14 | UP 按键 | 低电平有效 |
| P17 | 15 | DOWN 按键 | 低电平有效 |

三个按键可按以下方式连接：

- 按键一端接对应 PCF8575 输入
- 按键另一端接 GND
- 按下时输入为低电平

左转、右转、位置灯、近光灯、远光灯等车辆硬线信号必须先经过电平转换或隔离，不能把车辆电压直接送入 PCF8575。

## 8. MAX98357A 音频接线

音频功能为可选项。

| MAX98357A | ESP32 |
|---|---|
| BCLK | GPIO14 |
| LRC / WS | GPIO13 |
| DIN | GPIO25 |
| GND | GND |
| VIN | 按模块要求供电，常见模块使用 5V |

扬声器应接 MAX98357A 的 `SPK+`、`SPK-`，不要把扬声器任意一端接地。

固件上电时蓝牙音频默认关闭，可在设置页中开启。开启蓝牙音频时，WiFi 功能会关闭；重新开启 WiFi 时，蓝牙音频会停止。

## 9. 电源解锁与指示灯

### GPIO19 电源解锁输出

- 解锁状态：GPIO19 输出高电平
- 关机状态：GPIO19 输出低电平
- 默认密码：`000`

GPIO19 应连接外部驱动电路的控制端，例如 MOS 管驱动、三极管驱动或继电器模块输入端。

### GPIO2 WiFi 指示灯

- 有 WiFi 客户端连接时点亮
- 输出为高电平有效
- 外接 LED 必须串联限流电阻

## 10. 按键操作

| 按键 | 基本功能 |
|---|---|
| UP | 向上选择、数字加一；音乐页切换上一首 |
| DOWN | 向下选择、数字减一；音乐页切换下一首 |
| OK 短按 | 确认、进入选中页面、音乐播放/暂停 |
| OK 双击 | 返回或退出当前页面 |
| OK 长按 | 进入密码解锁；已解锁时执行关机 |
| KEY0 短按 | 开机并切换演示模式 |

补充说明：

- 密码输入时，UP/DOWN 修改当前数字，OK 确认并移动到下一位。
- 充电提示弹出时，短按 OK 可关闭当前提示。
- OTA 升级过程中不要退出、断电或重复操作。
- 本次运行中一旦开启过蓝牙，KEY0 会被屏蔽，重启设备后恢复。

## 11. 主要功能

固件当前主要提供以下功能：

- 240 × 240 彩色仪表显示
- 速度、挡位、电量、功率、里程和时间显示
- 转向灯、灯光、GPS、GSM、READY 等状态显示
- 边撑、TCS、巡航、电子座桶、HDC、充电、故障、低电量等状态提示
- 驾驶员未坐稳、能量回收、修复模式、ABS 等图标显示
- CAN 和 RS485 车辆数据接入
- PCF8575 硬线灯光和按键输入
- 充电状态提示
- 设置页亮度调节
- TCS、边撑感应和 HDC 设置
- 演示模式
- WiFi 与 Android 设备连接
- WiFi OTA 固件升级
- 可选蓝牙音频与 MAX98357A 功放输出

部分显示内容依赖车辆是否提供相应状态数据；未接车辆总线或没有对应数据时，相关数值可能显示 `--`，部分图标也不会点亮。

## 12. WiFi 使用

设备上电后默认开启 WiFi 热点：

| 项目 | 内容 |
|---|---|
| WiFi 名称 | `YADEA_485` |
| WiFi 密码 | `12345678` |
| 设备地址 | `192.168.4.1` |

WiFi 客户端连接后，GPIO2 指示灯点亮。

本固件的普通浏览器调试页面默认关闭；`192.168.4.1` 网页主要在进入 OTA 模式后用于固件升级。

## 13. OTA 升级方法

适用于设备已经运行本项目 OTA 版本的情况。

1. 在仪表设置页选择 `OTA`。
2. 手机或电脑连接 WiFi：`YADEA_485`。
3. 输入密码：`12345678`。
4. 浏览器打开 `http://192.168.4.1/`。
5. 选择提供的 `firmware-0.1.5.bin`。
6. 点击上传并等待进度完成。
7. 更新成功后设备会自动重启。

升级期间必须保持供电稳定，不要关闭网页、断开电源或重复上传。

## 14. 首次烧录说明

`firmware-0.1.5.bin` 是应用固件，最适合通过本项目已有的 OTA 页面升级。

如果是全新的空白 ESP32，仅有这个应用 `.bin` 通常不够，还需要匹配的：

- Bootloader
- 分区表
- OTA 数据分区
- 正确的烧录地址

本项目应用分区起始地址为 `0x10000`，使用 `partitions_ota_4mb.csv` 分区表。首次烧录建议使用完整工程通过 PlatformIO 烧录，或由固件提供方另外制作完整烧录包。不要把应用 `.bin` 直接写入地址 `0x0000`。

## 15. 上电前检查

上电前建议逐项确认：

- ESP32、屏幕、PCF8575、CAN 和 RS485 模块已经共地
- 车辆高压信号均已做电平转换或隔离
- ST7789 的 BL 已按当前定义接到 GPIO15
- CAN 已经过收发器，没有把 CANH/CANL 直接接到 ESP32
- RS485 模块电平与 ESP32 兼容
- GPIO19 后级有驱动电路，没有直接带负载
- I2C 地址为 `0x20`
- CANH/CANL、RS485 A/B 没有接反
- 总线终端电阻配置正确
- 电源电压、极性和供电能力符合各模块要求

如出现屏幕不亮、无法通信或反复重启，应先断电检查接线和供电，不要带电改线。
